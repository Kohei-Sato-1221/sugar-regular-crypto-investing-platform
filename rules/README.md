# Rules ディレクトリ

このディレクトリには、プロジェクトのルール、方針、ナレッジを蓄積します。Cursor AIが自動的にこれらのドキュメントを読み込み、コード生成や提案に活用します。

## ディレクトリ構成

```
rules/
├── README.md              # このファイル
├── testing-policy.md      # ユニットテスト方針
└── code-standards.md      # コーディング規約
```

## ドキュメント一覧

### テスト関連
- **testing-policy.md**: ユニットテストの方針、ルール、ベストプラクティス

### コーディング規約
- **code-standards.md**: コーディングスタイル、命名規則、エラーハンドリング

## 使い方

### Cursor AIとの連携
これらのドキュメントは、Cursor AIがコード生成や提案を行う際の参考資料として自動的に使用されます。

### ドキュメントの更新
- 新しいルールや方針が決まったら、適切なドキュメントに追加
- 既存のルールが変更されたら、該当ドキュメントを更新
- 変更履歴は各ドキュメントの冒頭に記載

## 追加予定のドキュメント

- アーキテクチャ設計
- デプロイメント手順
- トラブルシューティングガイド
- パフォーマンス最適化ガイド
